# Decisões Técnicas

Este documento registra as principais decisões tomadas durante o desenvolvimento do Tasky.

Cada decisão possui um identificador único (DE) para facilitar consultas futuras.

---

# DE-001 — Reduzir atritos na experiência do usuário

## Contexto

Durante o planejamento do projeto, foi definido que o Tasky deve evitar adicionar recursos que aumentem a complexidade sem gerar benefícios reais.

Essa decisão surgiu a partir da observação de sistemas utilizados no dia a dia, onde pequenas escolhas de interface acabam tornando tarefas simples mais demoradas e frustrantes.

## Decisão

Priorizar a experiência do usuário acima da quantidade de funcionalidades.

Toda nova funcionalidade deverá ser avaliada considerando seu impacto na usabilidade antes de ser implementada.

## Consequências

- Interfaces mais simples.
- Menos cliques desnecessários.
- Funcionalidades realmente úteis.
- Evolução baseada nas necessidades do usuário.
